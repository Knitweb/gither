The optional reasoning and benchmark layer for corpus import, query behavior, and reproducible RAG comparisons.

Start

gither repo-snapshot --repo /Users/develuse/repo/lens
gither route "benchmark Lens against LightRAG query output"

Concepts

  • Benchmarks must name corpus, task, expected answer, and evaluation method.
  • Query traces are evidence, not marketing claims.
  • Optional status means missing local checkout should not block unrelated workspace work.

Guides

  1. Corpus update Record source, snapshot time, and import command.
  2. Benchmark PR Attach baseline, candidate, and failed cases.
  3. Release Publish reproducible outputs before claiming model or retrieval gains.

Evidence

Accepted benchmark PRs should include fixtures, metrics, and reviewer notes that explain why the comparison is fair.

Recovery

If a benchmark is not reproducible, remove it from public claims and keep the failing trace for diagnosis.
