pulse

The live-state layer that records usage receipts, wallet activity, token events, and evidence that code was exercised.

Start

gither repo-snapshot --repo /Users/develuse/repo/pulse
PYTHONPATH=src python -m pytest -q

Concepts

  • Usage receipts are the bridge from reviewed code to observed demand.
  • Wallet and token events must stay verifiable without making a hosted server the trust root.
  • Gither enrichment can consume receipt counts as bounded evidence.

Guides

  1. Receipt change Update schema docs and add fixtures that prove the receipt shape.
  2. Wallet change Keep private control material outside public records.
  3. Gither handoff Export receipt counts only after the PR is merged and reviewed.

Evidence

Merged PRs should include receipt fixtures, test output, and the downstream enrichment export that consumed the receipts.

Recovery

If receipts drift, freeze enrichment imports, repair the schema or fixture, then rerun the score export.
