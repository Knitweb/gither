molgang

Game and node surface for 3D graph display, relay operations, and public 5mart deployment paths.

Start

gither repo-snapshot --repo /Users/develuse/repo/molgang
python -m pytest -q

Concepts

  • The browser UI is a display and node surface; it should not own truth.
  • 5mart routes must be checked after public page changes.
  • Graph interactions should map back to verifiable Knitweb or Gither records.

Guides

  1. UI change Verify desktop and mobile layout before mirroring.
  2. Relay change Test heartbeat and offline behavior.
  3. Release Check https://5mart.ml/molgang/ after publication.

Evidence

Merged PRs should include node health evidence, layout checks, and public route verification when the web surface changes.

Recovery

If 5mart display fails, check static assets first, then relay config, then the live node heartbeat.
