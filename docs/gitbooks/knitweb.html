knitweb

The relation fabric for content addressing, dependency records, and hashgraph-style navigation across the Knitweb stack.

Start

gither repo-snapshot --repo /Users/develuse/repo/knitweb
gither route "update dependency records for a shared module"

Concepts

  • Content IDs make records portable across mirrors.
  • Relations bind modules, releases, people, docs, and usage receipts.
  • Gither consumes the graph when ownership and enrichment need dependency evidence.

Guides

  1. Before editing Snapshot the repo, read the relation schema, and identify downstream repositories.
  2. During review Attach changed record types to the PR and include example CIDs.
  3. After merge Publish graph exports so Gither can verify dependency-aware claims.

Evidence

Accepted knitweb PRs should link dependency-record diffs, graph export checks, and any downstream Gither enrichment evidence.

Recovery

If a relation export is wrong, revert the graph artifact first, then repair ownership or usage records that consumed it.
