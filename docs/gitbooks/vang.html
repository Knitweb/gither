vang

Governance and optional issuer-backed authority for sensitive release actions, disputes, and time-series decision records.

Start

gither repo-snapshot --repo /Users/develuse/repo/vang
gither route "require issuer-backed release signing"

Concepts

  • Ordinary coding stays open; sensitive actions can require stronger attestations.
  • Issuer and registrar records should not leak private identity evidence into public docs.
  • Governance decisions must be portable enough for Gither to enforce during release.

Guides

  1. Issuer change Record who can issue, what they verify, and how revocation works.
  2. Release gate Link signer requirements to the exact Gither release operation.
  3. Dispute Preserve decision records without exposing private evidence.

Evidence

Merged PRs should include policy diffs, approval evidence, and the Gither command that consumes the policy.

Recovery

If a bad issuer rule ships, freeze sensitive releases, revert the rule, and publish a dispute note.
