knitweb-monitor

Repository telemetry and mirror health for GitHub, 5mart, and future Gither-controlled publication surfaces.

Start

gither repo-snapshot --repo /Users/develuse/repo/knitweb-monitor
python -m pytest -q

Concepts

  • Telemetry describes repository and mirror state; it is not the source of authority.
  • Health checks should be timestamped and tied to exact public URLs.
  • Gither uses monitor evidence to detect stale docs or failed mirrors.

Guides

  1. Add a check Define the URL, expected status, timeout, and failure text.
  2. Review a failure Confirm whether the source repo, mirror job, or hosting layer changed.
  3. Publish evidence Link monitor output from the operator playbook when a release depends on it.

Evidence

Accepted monitor PRs should include sample output for healthy and failing mirror states.

Recovery

When a mirror fails, verify the source docs first, then the publish job, then DNS or hosting state.
