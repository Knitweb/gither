Release and mirror operations

Gither operator playbook

This page is the checklist for shipping Gither-managed changes: prepare the local checkout, collect review evidence, merge through a PR, and mirror the static public pages to GitHub Pages and 5mart.

Daily flow Review checklist Release checklist Mirror checklist

Daily flow

  1. Confirm the target repo. Start in the checkout that owns the change, then check git status --short --branch.
  2. Create a narrow branch. Keep one operational concern per branch so the PR review stays readable.
  3. Make the smallest complete change. Update code, docs, tests, and public navigation together when they are one user-facing unit.
  4. Run local verification. Use the repository's normal tests plus gither gate when Python source is involved.
  5. Open a PR for review. Include commands run, public URL impact, and any mirror step still required.
  6. Merge only after review evidence is good. Hosted CI is useful, but Gither should also preserve portable review context.

Review checklist

  • The branch is based on current origin/main.
  • The diff contains only the intended feature, fix, or docs update.
  • Every new public page is linked from at least one stable entry point.
  • The README describes public URLs when the change affects publishing.
  • Static docs do not use inline scripts or click handlers.
  • Long-term authority is described as Gither/Knitweb records, not a hosted mirror.

Release checklist

git fetch origin
git status --short --branch
python3 -m compileall -q src tests
python -m pytest -q
PYTHONPATH=src python -m gither.cli gate --repo . --python-root src --strict
If a generated cache file appears during verification, remove it or ignore it before merging. Do not leave a dirty release checkout.

Mirror checklist

GitHub Pages

The Gither repo publishes from main/docs at https://knitweb.github.io/gither/.

5mart

The static mirror should be available at https://www.5mart.ml/gither/ after copying the docs output into the hosted www/gither path.

Independent target

https://gither.github.io/ is the desired identity once the gither account or organization is controlled.

Recovery flows

Problem Recovery
PR merged but GitHub Pages has old content. Check the Pages build, confirm the repo publishes from main/docs, and revalidate the exact page URL.
5mart mirror returns 404. Verify the gither/ directory exists under the live web root and contains index.html.
A reviewer asks what changed. Point to the PR diff, the change note if present, and the review pack when generated.
The repo is dirty after tests. Inspect generated files, keep intentional artifacts, and remove cache output before the final status check.
