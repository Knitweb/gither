GitHub tracks commits. Gither tracks value.

Gither is the ownership layer of the knowledge economy. It should feel familiar enough to adopt like GitHub, while moving review state, release state, and ownership state into portable records that can survive server lock-in.

The core product claim is not "agents write code." The stronger claim is that agentic work becomes reviewable, attributable, deployable, and eventually royalty-bearing across the Knitweb stack.

Operating Loop

  1. Route the request to the right repositories with gither route.
  2. Snapshot local state before editing with gither repo-snapshot.
  3. Use isolated worktrees or agent branches for parallel changes.
  4. Write change context into .gither/changes.
  5. Run gither python-audit and gither gate before acceptance.
  6. Emit release, mirror, and later usage-linked ownership records.
gither route "require issuer-backed release signing for vank agents"
gither repo-snapshot --repo .
gither change-note --summary "gate strong-identity release flow" --why "protect sensitive operator authority"
gither gate --repo . --python-root src

Example Case: Knitweb + Gither + Vank

  1. A maintainer wants releases that affect agent identity or treasury control to require stronger accountability than a normal device key.
  2. Gither routes the task across gither, vank, and the relevant protocol repository, then records the accepted change notes and review receipts.
  3. Knitweb records the dependency relation: the release gate depends on issuer and registrar records, plus the identity attestation boundary.
  4. Vank provides the optional attestation rail: public records stay PII-free, but a release signer can still be traced through an encrypted evidence path when policy requires it.
  5. Pulse later records real use. If the feature is deployed and used, the resulting value flow can follow reviewed dependency relationships rather than raw commit counts.

Why This Beats the Hosted End State

  • GitHub is familiar, but it is still a hosted trust root and a Microsoft lockout risk.
  • GitLab is not the answer to that problem. It still depends on servers, operators, hosted instances, and platform uptime.
  • Radicle already proves the sovereign-forge direction, but Gither can go further on review gates, deploy receipts, ownership, and optional identity-backed authority.
  • That is why Gither should first be sold as a p2p GitHub duplicate, then grown into a marketplace for living software.
